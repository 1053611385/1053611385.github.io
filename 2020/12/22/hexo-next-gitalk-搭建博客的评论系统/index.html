hexo + next + gitalk 搭建博客的评论系统

发表于 2020-12-22

前言

关于通过hexo + next + gitalk的文章有很多，但是大部分都有些过时了，现在的next的一些文件格式已经从swig改为了njk，他们之间的语法有些不同，所以我进行了验证之后，根据之前的文章做了一下新的梳理，大体还是之前的流程，但是有一些细微的差别

Gitalk + OAuth

Gitalk

如果想要在自己的博客系统添加评论系统，是需要依赖第三方的系统能力的，比如disqus，livere，gitalk等都是第三方的评论系统，我们今天只说Gitalk。

Gitalk是Github提供给我们的一个免费的工具，他是基于issue 和 preact(一句话介绍Preact，它是React的3KB轻量替代方案，拥有同样的ES6 API)；它的原理是，我们在github创建一个仓库，这个仓库是任意的，然后github提供一个js文件，我们在我们的博客系统中，将js文件加载进来，同时在主题的模板中配置div，把评论的界面添加进来；文章界面初始化的时候会将Gitalk的界面显示出来，js文件加载出来，然后每一个文章会在github仓库中添加一个issue，所有的评论需要先登录github，然后通过Oauth授权，进行文章的评论

Oauth

Oauth是一个网络授权标准，现在已经是2.0版本，理解OAuth 2.0，可以通过第三方系统来授权，帮助你的系统对用户进行一些登录认证，但是又不会威胁用户的信息安全，你是获取不到用户的密码的。

申请Gitalk

登录github，进行Gitalk权限申请

参数说明

  • Application name： # 应用名称，随意
  • Homepage URL： # 网站URL，如https://xjt927.github.io
  • Application description # 描述，随意
  • Authorization callback URL：# 网站URL，https://xjt927.github.io

如果你的Github Pages已经绑定了域名，这两个参数Homepage URL、Authorization callback URL需要写你自己的域名。

注册成功后会得到Client ID、Client Secret 两个参数。
